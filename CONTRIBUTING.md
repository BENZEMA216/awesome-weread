# Contributing to Awesome WeRead

Thanks for keeping this list useful.

## Adding an entry

1. Find the right section. Don't create a new section unless there are 2+ entries that don't fit.
2. Use this format: `[owner/repo](https://github.com/owner/repo) — Short factual description (≤ 100 chars).`
3. Keep descriptions specific. "WeRead tool" is useless. "Sync WeRead highlights to Notion via Cloudflare Worker" is useful.
4. For Agent Skills / MCP servers, name the runtime: Claude Code, OpenClaw, Hermes, Cursor, ChatGPT, etc.
5. Sort within a section by relevance / popularity, not alphabetically. Best-known and most-actively-maintained at the top.

## Quality bar

A project belongs here if it meets **all** of:

- Public, with a README that explains what it does.
- Working — last commit within ~12 months OR clearly marked stable.
- Not a stale fork or a vendored mirror of something already listed.
- Solves a real WeRead use case (read / sync / analyze / extend / export / automate).

Reject:

- Generic Chinese book-reading tools that happen to mention 微信读书 in passing.
- Throwaway course projects or "hello world" demos.
- Anything that violates WeRead's TOS in a way the author hasn't acknowledged (e.g., undisclosed scrapers presented as official tools).

## Removing or moving an entry

PRs to remove archived / broken projects are welcome. Include a one-line reason in the PR body (e.g., "404 since 2026-04, see commit log").
