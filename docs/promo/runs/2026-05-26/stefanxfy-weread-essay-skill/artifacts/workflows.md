# Workflow：读后感写作（essay）

## Step 1 — 确认书目和范围

用户输入可能包含：

- 书名（精确或模糊）
- 章节名（可选）
- 背景材料链接或文件路径（可选，若无则提示用户提供）
- 个人笔记说明（可选，用户可能已粘贴划线内容）

**优先从用户输入中提取**。若用户只说「帮我写一篇《书名》的读后感」，则按以下步骤处理：

1. 用 `/store/search` 确认书名对应的 bookId
2. 调 `/book/bookmarklist` 确认该书/章节有无划线
3. 若无划线，告知用户：「这本书里还没有划线记录，无法生成基于个人笔记的读后感。你是否想先读一遍、划一些句子，然后我们再来写？」

若用户提供了背景材料链接/文件，读取并理解其内容（故事脉络、人物关系、核心议题）。

## Step 2 — 获取微信读书数据

**同时调用**：

```
/book/bookmarklist?bookId={bookId}        # 获取该书全部划线
/review/list/mine?bookid={bookId}&count=100  # 获取个人想法/点评
```

若指定了章节（非全书），用 `/book/chapterinfo` 获取章节列表，找到对应的 `chapterUid`，在展示划线时过滤到该章节。

**合并输出**：划线按章节分组，笔记感想单独标注，汇总后告诉用户：

- 「这本书共有 X 条划线，X 条想法/点评」
- 「其中第 X 章（章节名）有 X 条划线」
- 若有背景材料链接：「已读取背景材料，故事脉络/核心议题已了解」

## Step 3 — 与用户确认写作方向

**检查点：必须在正式开始写作前确认**

> 我已拉取了你在这本书/这章的 X 条划线和 X 条笔记感想。如果提供了背景材料我也已阅读。
>
> 正式开始之前，想确认几件事：
>
> 1. **正文的核心论点**——根据你的划线内容，我预判可以展开这几个方向：
>    - 方向 A（你的哪条划线印证了这一点）
>    - 方向 B（……）
>    你最想深入哪 2-3 个？还是全要？
>
> 2. **附录的范围**——是否全部划线都保留进附录？还是你有想特别删掉或特别保留的？
>
> 3. **背景材料**——你提供了 `链接/文件路径`，我会在生成正文时作为故事脉络和人物关系的参考，不会在正文里大段复述。如果没有提供背景材料，我会直接从你的划线内容来推断核心议题，但判断可能不够准确。

**无人值守模式 fallback**：如果检测到当前调用没有交互通道（自动化流水线等），跳过检查点，直接按默认值继续：

- 默认展开 3 个核心论点
- 附录保留全部划线
- 若无背景材料，直接从划线推断核心议题

并在输出末尾注明：「（无人值守模式，按默认参数生成。如需调整，告诉我重新写。）」

## Step 4 — 生成正文

按以下结构写（**不拘泥于这个顺序，允许自由切入**）：

1. 一个有吸引力的开头——从你的某条划线、某个想法、或者一个直抵核心的问题切入
2. 2-4 个核心论点展开，每个论点：
   - 从背景材料或划线内容中生长出来
   - 结合你的个人感想，说明「为什么这句话让我停下来」
   - 必要时用 `/store/search` 补充公开讨论中的有趣视角
   - 禁止大段摘抄，最多 1 处不超过三行的引用
3. 1-2 个发人深省的结论——不是总结，是读者会反复想的那句话

**写作风格守则**：

- 文学性 + 理性分析 + 情感抒发三者穿插，不堆砌
- 不用「首先、其次、最后、综上所述」
- 不用破折号（全文 ≤ 1 处）
- 不用书评腔、学术腔、鸡汤腔
- 像一个人在深夜和你聊他刚读完的一篇改变了他的东西

## Step 5 — 生成附录

格式固定，如下：

```
## 附：我划下的每一句话

以下是我在这个章节/这本书里划下的全部句子，每条后面附有我当时的即时感想或事后反思。

1. 「……原文摘录第一句……」
   我的感受/思考：（可以是几个字，也可以是一段，诚实为上）

2. 「……原文摘录第二句……」
   我的感受/思考：……

（以此类推，全部保留，不删减）
```

- 划线原文**全部保留**，一条都不删
- 感想字数不设下限——有的几个字，有的写一段
- 感想内容类型：划线时的感受、划完后的新想法、和正文论点的呼应、对某个词的困惑、读到此处想起的另一本书……都可以

## Step 6 — 预览并交付

输出完整内容（正文 + 附录），在正文开头注明书名、章节、划线条数。

末尾注明：

> 以上是为你生成的读后感草稿。如需调整某个论点的深度、附录的侧重、或者整体语气，告诉我。
